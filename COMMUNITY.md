## Getting Started

### For Users

1. **Read the docs**: Start with [README.md](README.md) and [QUICKSTART.md](QUICKSTART.md)
2. **Try examples**: Explore [examples/](examples/) directory
3. **Check FAQ**: Look for answers in [docs/FAQ.md](docs/FAQ.md)
4. **Ask questions**: Use [GitHub Discussions](https://github.com/ananke-ai/ananke/discussions) if you need help

### For Contributors

1. **Read [CONTRIBUTING.md](CONTRIBUTING.md)**: Contribution guidelines and development setup
2. **Check [issues](https://github.com/ananke-ai/ananke/issues)**: Find issues labeled `good first issue` or `help wanted`
3. **Review [ARCHITECTURE.md](docs/ARCHITECTURE.md)**: Understand the system design
4. **Set up development environment**: Follow [Development Setup](CONTRIBUTING.md#development-setup)

## Ways to Contribute

### Code Contributions

- **Bug fixes**: Fix reported issues and add regression tests
- **Features**: Implement new functionality (see [CONTRIBUTING.md](CONTRIBUTING.md))
- **Performance**: Optimize hot paths and add benchmarks
- **Testing**: Improve test coverage and add edge case tests
- **Refactoring**: Improve code clarity and maintainability

### Documentation

- **User guides**: Help us write better tutorials
- **API docs**: Improve API documentation with examples
- **Examples**: Contribute working examples for common use cases
- **Blog posts**: Share how you're using Ananke

### Community Support

- **Help others**: Answer questions in [GitHub Discussions](https://github.com/ananke-ai/ananke/discussions)
- **Report bugs**: Submit detailed bug reports with reproductions
- **Suggest features**: Propose enhancements with clear use cases
- **Give feedback**: Help us understand your needs

### Language Support

Help us add support for more languages:

- **Currently supported**: TypeScript, Python, Rust, Zig, Go
- **Next targets**: Java, C++, C#, Ruby
- **Your suggestion**: Have a language in mind? [Start a discussion](https://github.com/ananke-ai/ananke/discussions)

## Contributing Your First PR

1. **Find an issue**: Look for `good first issue` or `help wanted` labels
2. **Comment on the issue**: Let us know you want to work on it
3. **Fork and create branch**: `git checkout -b fix/issue-123`
4. **Make changes**: Write code and tests
5. **Follow guidelines**: See [CONTRIBUTING.md](CONTRIBUTING.md)
6. **Submit PR**: Link to the issue in your PR description
7. **Respond to feedback**: We'll review and suggest changes
8. **Celebrate**: Your first contribution!
